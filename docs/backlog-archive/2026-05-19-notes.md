## 7e

**Reverted 2026-05-18.** All three Vercel projects flipped back to *Settings → Build and Deployment → Ignored Build Step → **Automatic*** (every push rebuilds, no skip logic).

### Why reverted

Both attempts at a skip script caused silent build cancellations that cost more debugging time than they saved.

- **Attempt 1** (`git diff --quiet HEAD^ HEAD -- <paths>`) — broke on multi-commit pushes. When the latest commit was docs-only, `HEAD^` pointed at the docs commit's parent and the source-code commit's changes lived at `HEAD^^`, outside the diff window → silent skip.
- **Attempt 2** (`git diff --quiet "${VERCEL_GIT_PREVIOUS_SHA:-HEAD^}" HEAD -- <paths>`) — silently cancelled ALL three projects on a push that legitimately touched two of them. Root cause not diagnosed; either Vercel's `VERCEL_GIT_PREVIOUS_SHA` semantics differ from documented, or the shallow clone doesn't have the previous SHA available, or the shell parameter expansion isn't behaving as expected. Eating ~30 s of build time per push was a smaller cost than continuing to debug.

### What was tried, kept for reference

- **Site** (`kol-client-acyr-website`): `git diff --quiet "${VERCEL_GIT_PREVIOUS_SHA:-HEAD^}" HEAD -- apps/website packages assets pnpm-lock.yaml pnpm-workspace.yaml`
- **Studio** (`kol-client-acyr-studio`): `git diff --quiet "${VERCEL_GIT_PREVIOUS_SHA:-HEAD^}" HEAD -- apps/studio assets pnpm-lock.yaml pnpm-workspace.yaml`
- **Styleguide** (`kol-client-acyr-styleguide`): `git diff --quiet "${VERCEL_GIT_PREVIOUS_SHA:-HEAD^}" HEAD -- apps/styleguide apps/website/src/components apps/website/src/data packages assets pnpm-lock.yaml pnpm-workspace.yaml`

### If revisited

- Diagnose via the Build Logs / Ignored Build Step output on a Canceled deployment to see the actual command stdout/stderr/exit code.
- Consider a Vercel build hook script in `.vercel/` instead of an inline shell expression — easier to log and test locally.
- Sanity-check that Vercel's shallow clone depth includes the previous SHA before relying on it.

## 19. Form-element chrome reset — repo-wide
- ▢ Native browser chrome bleeds through on form controls across the repo — `<input>` / `<select>` / `<textarea>` show default borders, focus rings, accent colors that don't match the DS. Concrete trigger: ContentFilters' search field shows blue strokes above/below the input when expanded (visible in styleguide Gallery, likely also on live `/shop` filter in dark mode). Tailwind 4 doesn't include `@tailwindcss/forms`; nothing in `packages/ds/` resets these. Fix: add a global base layer in `packages/ds/framework/framework.css` (or a new `packages/ds/utilities/forms.css`) that resets `input/select/textarea` border, outline, background, appearance, and accent-color to DS tokens. Related: 9f text-selection quirks — same family of "browser default vs DS" friction.

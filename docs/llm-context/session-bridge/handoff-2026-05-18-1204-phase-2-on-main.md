## Working memory not yet in AGENT-CONTEXT

- **The Phase 1 → Phase 2 editor bug is a lesson worth remembering.** Phase 1 explicitly excluded editor-internal CSS files (kol-editor.css, kol-typography-fonts-full.css) from the bulk rename because they were tagged for Phase 4. But Phase 1 DID rename the JSX class names that reference those CSS rules. That created a silent mismatch — build green, no errors, layout silently collapsed. Lesson: when scoping a partial rename, EITHER rename the CSS classes AND the JSX consumers together, OR rename neither — never one without the other. The bulk JSX sed in Phase 1 was too broad; it should've scoped to JSX referencing CSS rules that WERE being renamed.
- **`includeFiles` glob behavior on Vercel:** literal paths (`"dist/app.html"`) work when matching one file. Globs like `"dist/**"` work but bundle everything matching, including unrelated static assets (in our case the 305 MB `public/brand/` tree gets copied into `dist/` by Vite). Always start with the narrowest glob possible; the function size limit is 300 MB and Vite-copied public assets explode it fast.
- **Multi-candidate path resolution in serverless functions is worth doing by default.** `process.cwd()` and `import.meta.url` resolve to different things in different Vercel runtime configurations. The proxy's three-path try-each-in-order pattern is now the template for any function that reads bundled files — copy it forward if we add more.
- **`@components` + `@website-data` aliases in styleguide** are temporary. Phase 4 retires both: JSX extraction kills `@components`, dead-data deletion kills `@website-data`. Don't add new code that depends on either alias.
- **pnpm v11 `allowBuilds` syntax:** the OLD syntax `onlyBuiltDependencies: [...]` no longer works alone; v11 wants `allowBuilds.<pkg>: true`. The install errors fast with `ERR_PNPM_IGNORED_BUILDS` if you miss this — and the workspace-yaml linter helpfully injects an `allowBuilds: <pkg>: set this to true or false` placeholder for you to complete.
- **Vercel function bundler does NOT auto-include files referenced via runtime `readFile()`** — only files reached via static `import`/`require`. Any function that reads a template at runtime needs `functions.<path>.includeFiles` in vercel.json.
